## 1. Platform & Mission

- **Domain:** Persistent agent OS — 24/7 Claude Code sessions coordinating via a durable priority queue, isolated by git worktrees, surfaced through Telegram + web dashboard.
- **Tech stack:**
  - **Runtime:** TypeScript + Node.js 22 (ESM modules, `moduleResolution: bundler`).
  - **Queue + memory:** `better-sqlite3` (default) / PGLite / Postgres / Cloudflare D1 behind `QueueEngine`.
  - **Process supervision:** PM2 (`ecosystem.config.js` auto-generated by `cortextos ecosystem`).
  - **Agent runtime:** `node-pty` spawning `claude` CLI — Keychain (macOS) or `CLAUDE_CODE_OAUTH_TOKEN` (headless).
  - **Dashboard:** Next.js 16 (React 19) + Tailwind v4 + shadcn + `@base-ui/react`, monochrome Manrope.
  - **Testing:** vitest (unit), playwright (e2e).
  - **Build:** tsup.
- **Goal:** Ship a distributable, durable, priority-sequenced agent platform (SOMA) that can be instantiated against any business's digital twin.
- **Full vision:** `PROJECT_SOMA.md`. Always read first when resuming a session.

## 3. The Harness — Execution Hard-Limits

These stop agent-drift.

- **Bug fixes:** ≤ 50 lines changed per session. One fix = one commit.
- **New features:** ≤ 300 lines per session. Split across commits if larger.
- **File hygiene:** ≤ 500 lines per source file. Extract to helpers or submodules if exceeded.
- **Port commits from gbrain/gstack** are exempt from the line ceiling — preserving capability parity per ADR-011 — but must annotate `// SOMA:` inline for every deviation.
- **Validation mandatory before "done":** every task ends with `npx tsc --noEmit` output shown + relevant test suite passing. No silent claims of success.
- **High-stakes paths** (auth, payments, PTY spawn, OAuth, shell handler, schema migrations): Human-in-the-Loop confirmation required before landing. Do not push directly without flagging.

## 5. Data & Durability Discipline

- **Timestamps:** SOMA stores Unix ms (number) in every DB row. Display converts via `new Date(ms)` at the boundary. Never mix `Date` and `number` across an API.
- **Quiet hours / crons:** IANA timezone only (`America/Los_Angeles`, not `PST`). Wall-clock evaluation uses `Intl.DateTimeFormat`; see `src/minions/quiet-hours.ts`.
- **Append-only invariants:** `minion_jobs.stacktrace`, `minion_inbox`, `minion_rate_leases`, future brain `learnings` — append-only. Never `DELETE` raw records; mark with `released_at` / `read_at` / `deleted_at` columns.
- **Rate leases:** every Anthropic-hitting job MUST `acquireLock` on a scope before calling the API. Release on job completion (any terminal state). See `engine.acquireLock` contract.
- **Schema changes:** ship with a migration SQL + a schema-version bump. Never ALTER TABLE behind feature flags.

## 6. Environment & Security

- **Secret masking:** `.env` files are readable for context but NEVER echo full values. Mask via `sed -E 's/=(.{0,4}).*/=\1.../'` or similar before printing. Same rule for Bot tokens, OAuth tokens, API keys in logs.
- **No-commit rule:** never `git add` `.env`, `*.json` credentials, `.cortextos-env`, or `orgs/**/secrets.env`. Check `.gitignore` before any `git add` broader than file-by-file.
- **Boundary checks:** never modify `src/pty/agent-pty.ts` env allowlist, `src/cli/install.ts` token plumbing, or the Telegram ALLOWED_USER gate without Human-in-the-Loop review. These are security surfaces.
- **OAuth:** Claude Code OAuth lives in macOS Keychain under `"Claude Code-credentials"`. For headless: `claude setup-token` generates `CLAUDE_CODE_OAUTH_TOKEN`. Never log the token. Never commit an env file containing it.

## 9. Database & Timezone Rules

- **Schema source of truth:** `src/minions/schema.sql`. Dashboard-specific SQLite stores live elsewhere; Minions is canonical.
- **SQLite PRAGMAs on open:** `WAL` + `synchronous=NORMAL` + `foreign_keys=ON` + `busy_timeout=5000`. Set automatically by `openSqliteEngine`; do not reconfigure ad-hoc.
- **Advisory locks:** `BEGIN IMMEDIATE` + sentinel row in `minion_rate_leases` with `owner_job IS NULL`. Postgres engine (Phase 7) restores `pg_advisory_xact_lock`.
- **Timestamps:** Unix ms stored; `unixepoch('subsec') * 1000` as SQLite default. Never `CURRENT_TIMESTAMP` (returns text).
- **JSON fields:** TEXT columns containing JSON.stringify output. Parse at read boundary via `rowTo*` helpers in `types.ts`.
